# Story 1.1: Configuração do Monorepo e Ambiente Docker

## Status
Approved

## Story
**As a** Desenvolvedor,
**I want** um monorepo pré-configurado com **Nx** e Docker Compose,
**so that** eu possa configurar meu ambiente de desenvolvimento local completo com um único comando.

## Acceptance Criteria
1. O monorepo é inicializado usando Nx.
2. A estrutura de diretórios `apps/`, `packages/`, `docs/`, `.docker/` é criada na raiz do projeto.
3. Um arquivo `docker-compose.yml` é criado na raiz.
4. O comando `docker-compose up --build` executa com sucesso, sem erros.
5. O backend (Litestar) e o frontend (Next.js) são definidos como serviços no `docker-compose.yml`.

## Tasks / Subtasks
- [ ] Inicializar o monorepo com Nx. (AC: 1)
- [ ] Criar a estrutura de diretórios base (`apps`, `packages`, `docs`, `.docker`). (AC: 2)
- [ ] Criar o arquivo `docker-compose.yml` na raiz do projeto. (AC: 3)
- [ ] Definir o serviço do backend Litestar no `docker-compose.yml`. (AC: 5)
- [ ] Definir o serviço do frontend Next.js no `docker-compose.yml`. (AC: 5)
- [ ] Garantir que os Dockerfiles para backend e frontend sejam criados e referenciados corretamente.
- [ ] Validar a execução do `docker-compose up --build`. (AC: 4)

## Dev Notes
- **Arquitetura:** A arquitetura do projeto é um monorepo gerenciado com Nx, com um frontend Next.js e um backend Python (Litestar), containerizado com Docker. [Fonte: docs/architecture.md#1-arquitetura-de-alto-nivel]
- **Stack de Tecnologia:**
    - Monorepo: Nx (~18.2)
    - Container: Docker
    - Backend: Python ~3.11, Litestar ~2.9
    - Frontend: Next.js ~14.1, TypeScript ~5.3
    [Fonte: docs/architecture.md#2-stack-de-tecnologia]
- **Estrutura do Projeto:**
    ```
    boilerplate-agente-ia/
    ├── apps/
    │   ├── backend/
    │   └── frontend/
    ├── packages/
    │   ├── shared/
    │   └── ui/
    ├── docs/
    ├── .docker/
    ├── agente.md
    ├── docker-compose.yml
    ├── package.json
    └── nx.json
    ```
    [Fonte: docs/architecture.md#9-estrutura-unificada-do-projeto-monorepo]
